# nr-mqtt Architecture

nr-mqtt is a Docker Compose application that bridges MQTT telemetry from
Particle Photon devices to the New Relic Event API.

---

## Directory Structure

```
nr-mqtt/
├── docker-compose.yml          # Defines mosquitto + relay services
├── .env                        # Secret config (gitignored)
├── .env.example                # Template — safe to commit
├── mosquitto/
│   ├── config/
│   │   └── mosquitto.conf      # Broker configuration
│   └── passwd                  # Hashed password file (gitignored)
└── relay/
    ├── Dockerfile
    ├── package.json
    ├── package-lock.json
    └── src/
        └── index.js            # Relay service entrypoint
```

## Docker Compose Services

### `docker-compose.yml`

```yaml
version: "3.9"

services:
  mosquitto:
    image: eclipse-mosquitto:2
    restart: unless-stopped
    ports:
      - "1883:1883"       # MQTT plain TCP (LAN only — do not expose to internet)
    volumes:
      - ./mosquitto/config:/mosquitto/config:ro
      - ./mosquitto/passwd:/mosquitto/passwd:ro
      - mosquitto_data:/mosquitto/data
      - mosquitto_log:/mosquitto/log
    healthcheck:
      test: ["CMD", "mosquitto_sub", "-t", "$$SYS/#", "-C", "1", "-i", "healthcheck", "-W", "3"]
      interval: 30s
      timeout: 10s
      retries: 3

  relay:
    build: ./relay
    restart: unless-stopped
    depends_on:
      mosquitto:
        condition: service_healthy
    environment:
      - NEW_RELIC_ACCOUNT_ID=${NEW_RELIC_ACCOUNT_ID}
      - NEW_RELIC_INSERT_KEY=${NEW_RELIC_INSERT_KEY}
      - MQTT_BROKER_URL=${MQTT_BROKER_URL}
      - MQTT_USERNAME=${MQTT_USERNAME}
      - MQTT_PASSWORD=${MQTT_PASSWORD}
    healthcheck:
      test: ["CMD", "node", "-e", "require('http').get('http://localhost:3000/health', r => process.exit(r.statusCode === 200 ? 0 : 1))"]
      interval: 30s
      timeout: 5s
      retries: 3

volumes:
  mosquitto_data:
  mosquitto_log:
```

## Mosquitto Configuration

### `mosquitto/config/mosquitto.conf`

```
# Listener
listener 1883
protocol mqtt

# Disable anonymous access — all clients must authenticate
allow_anonymous false

# Password file (bcrypt hashes generated by mosquitto_passwd)
password_file /mosquitto/passwd

# Persistence
persistence true
persistence_location /mosquitto/data/

# Logging
log_dest stdout
log_type error
log_type warning
log_type notice
log_type information
```

Key points:

- `allow_anonymous false` — every client must supply a username and password.
- `password_file` — bcrypt-hashed credentials managed by `mosquitto_passwd`.
- Port 1883 is plain TCP. Do not expose this port to the internet; restrict it
  to the local LAN via firewall rules or Docker host network policy.
- No TLS listener is configured because the `hirotakaster/MQTT` Particle library
  does not support TLS. See gee-force `ARCHITECTURE.md` for mitigation details.

## Provisioning Device Credentials

Add a new device account to the password file with `mosquitto_passwd`. The
`-b` flag supplies the password non-interactively (suitable for scripting):

```bash
# Create the passwd file and add the first user (-c creates the file)
docker run --rm -v $(pwd)/mosquitto:/mosquitto eclipse-mosquitto:2 \
  mosquitto_passwd -b -c /mosquitto/passwd geeforce-device changeme

# Add a second user without -c (appends)
docker run --rm -v $(pwd)/mosquitto:/mosquitto eclipse-mosquitto:2 \
  mosquitto_passwd -b /mosquitto/passwd geeforce-device2 anotherpassword

# Reload broker config (no restart needed for passwd changes in Mosquitto 2.x)
docker compose kill -s SIGHUP mosquitto
```

The `passwd` file is gitignored. Back it up separately alongside your `.env`.

## Relay Service Design

### Runtime

Node.js 20 (LTS). Dependencies: `mqtt` (MQTT client), `node-fetch` (NR HTTP
POST). No framework required.

### `relay/Dockerfile`

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY src/ ./src/
EXPOSE 3000
CMD ["node", "src/index.js"]
```

### `relay/src/index.js` — design summary

```
startup
  read env vars (validate required vars, exit 1 if missing)
  connect to Mosquitto with mqtt.connect(MQTT_BROKER_URL, { username, password })
  subscribe to geeforce/+/accel (QoS 0)
  start flush timer (2000ms interval)
  start HTTP health server on :3000

on MQTT message (topic, payload)
  parse JSON payload
  map short fields to NR event fields:
    ts  → timestamp
    x   → accelX
    y   → accelY
    z   → accelZ
    m   → magnitude
    dev → deviceId
  set eventType = "AccelSample"
  push mapped event to pendingBatch[]
  if pendingBatch.length >= 500: flushBatch()

flushBatch()
  if pendingBatch is empty: return
  snapshot = pendingBatch.splice(0)       // drain atomically
  POST snapshot to NR Event API (up to 3 retries, 1s delay between)
  log result

NR Event API call
  POST https://insights-collector.newrelic.com/v1/accounts/{accountId}/events
  Headers:
    Content-Type: application/json
    X-Insert-Key: {NEW_RELIC_INSERT_KEY}
  Body: JSON.stringify(snapshotArray)

health server GET /health
  returns 200 OK {"status":"ok","pending":N,"mqttConnected":bool}
  returns 503 if MQTT is not connected
```

### Batching behaviour

| Condition            | Action          |
|----------------------|-----------------|
| `pendingBatch.length >= 500` | Flush immediately |
| 2-second timer fires | Flush if batch non-empty |
| POST fails (≤3 attempts) | Retry with 1s delay |
| POST fails (>3 attempts) | Discard batch, log error |

At 3 Hz per device the 2-second window produces ~6 events per flush. The
500-event cap is a safety valve for burst scenarios or multiple simultaneous
devices.

## Environment Variables

| Variable               | Required | Description                                      |
|------------------------|----------|--------------------------------------------------|
| `NEW_RELIC_ACCOUNT_ID` | Yes      | Numeric NR account ID                            |
| `NEW_RELIC_INSERT_KEY` | Yes      | NR Ingest/Insert API key (`NRII-...`)            |
| `MQTT_BROKER_URL`      | Yes      | Broker URL, e.g. `mqtt://mosquitto:1883`         |
| `MQTT_USERNAME`        | Yes      | MQTT username for relay service                  |
| `MQTT_PASSWORD`        | Yes      | MQTT password for relay service                  |

### `.env.example` (committed to git)

```dotenv
NEW_RELIC_ACCOUNT_ID=your_account_id_here
NEW_RELIC_INSERT_KEY=NRII-your_insert_key_here
MQTT_BROKER_URL=mqtt://mosquitto:1883
MQTT_USERNAME=nr-relay
MQTT_PASSWORD=changeme
```

### `.env` (gitignored — real secrets)

Copy `.env.example` to `.env` and fill in real values:

```bash
cp .env.example .env
$EDITOR .env
```

`.gitignore` must include:

```
.env
mosquitto/passwd
```

## Secret Management

- All secrets live in `.env` on the Docker host filesystem.
- Docker Compose injects them as environment variables into the relay container.
- The New Relic Insert Key is never written to disk inside the container and
  never appears in MQTT messages or broker logs.
- The relay has a separate MQTT credential (`nr-relay`) distinct from the device
  credential (`geeforce-device`). This limits blast radius if one credential is
  compromised.
- Rotate credentials by updating `.env` and `mosquitto/passwd`, then running
  `docker compose restart`.

## Health Checks

### Mosquitto

Docker Compose health check subscribes to `$SYS/#` and waits for one message.
The `$SYS` topic tree is published by Mosquitto every 10s and provides broker
statistics. A successful receive confirms the broker is accepting connections.

### Relay

The relay exposes `GET /health` on port 3000 (internal only, not mapped to the
host). Docker Compose runs `node -e "..."` to issue a local HTTP GET. The
endpoint returns 200 if the relay is running and MQTT is connected, 503
otherwise. `depends_on: condition: service_healthy` ensures the relay does not
start until the broker is healthy.

## Operational Runbook

### Start the stack

```bash
cd /path/to/nr-mqtt
cp .env.example .env && $EDITOR .env          # first time only
docker compose up -d
docker compose logs -f                        # tail logs
```

### Verify data flow

```bash
# Subscribe as a test client to watch live messages
mosquitto_sub -h localhost -p 1883 \
  -u geeforce-device -P changeme \
  -t 'geeforce/#' -v

# Check relay health
curl http://localhost:3000/health
```

### Add a new device credential

```bash
docker run --rm -v $(pwd)/mosquitto:/mosquitto eclipse-mosquitto:2 \
  mosquitto_passwd -b /mosquitto/passwd <new-username> <new-password>
docker compose kill -s SIGHUP mosquitto
```

### Rotate the New Relic Insert Key

```bash
$EDITOR .env                    # update NEW_RELIC_INSERT_KEY
docker compose restart relay
```
